# 安全政策 (Security Policy)

## 报告安全漏洞

如果你发现 Claude Builders Bounty 中存在安全漏洞，请**不要**在公开的 GitHub Issues 中报告。

### 如何报告

请发送电子邮件至：**security@claudebounty.dev**

在邮件中请包含：
- 漏洞的详细描述
- 受影响的组件或文件
- 潜在影响和严重程度
- 建议的修复方案（如果有）

### 报告规范

- ✅ **尽快通知我们** - 发现后立即报告
- ✅ **包含必要细节** - 帮助我们快速理解问题
- ✅ **给予合理时间** - 通常我们会在 7 天内回复
- ❌ **不要公开披露** - 等待我们的修复和声明
- ❌ **不要利用漏洞** - 仅用于验证和演示

### 响应流程

1. **确认** (1-2 天) - 我们会确认收到你的报告
2. **评估** (3-5 天) - 分析漏洞严重程度
3. **修复** (5-10 天) - 开发和测试补丁
4. **发布** - 合并修复并发布更新
5. **信息** - 向报告者和社区通知

## 安全最佳实践

### 对于赏金发布者

- ✅ 清晰描述验收标准
- ✅ 使用环境变量管理敏感信息
- ✅ 定期审查拉取请求
- ✅ 验证贡献者身份
- ❌ 不要在 Issue 中暴露 API 密钥或凭证

### 对于赏金领取者

- ✅ 遵循 CODE_OF_CONDUCT.md
- ✅ 编写安全的、可审查的代码
- ✅ 测试你的解决方案
- ✅ 在 PR 中清晰描述更改
- ❌ 不要提交未审查的第三方代码
- ❌ 不要故意引入安全问题
